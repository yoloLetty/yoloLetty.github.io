    安装 Hexo

  • 首先创建一个文件夹（确保所在空间足够大），这个文件夹用来存在 blog 的东西
  • 可用 cmd 命令 / blog 文件夹的页面 shift+右击 选择 ”在此打开powershell窗口“ / Git Bash here ，三种方式来安装 Hexo


  • Hexo 全局安装命令：npm i -g hexo/npm install -g hexo

    1. 也有教程写的安装命令是：npm install -g hexo-cli / npm install hexo -save（但不要先使用这个命令！）

    2. hexo vs hexo-cli

      hexo 具备的功能封装为命令，提供给用户通过 hexo server / hexo deploy 等命令调用的模块，就是hexo-cli (CLI = Command Line Interface)

  • 安装完成后，查看版本 hexo -v，确认是否安装成功；如果显示结果如下图则正确

  • hexo g 生成 public 文件夹 （该文件夹下为 .md 文件编译后生成的静态文件，包括 html/css/js 图片和 db.json 文件，也就是被 hexo deploy 部署到 GitHub 上的文件）

3. 添加 SSH Key 到 GitHub

有两种方法可以找到 SSH Key

  1. 找到生成的 .ssh 的文件夹中（路径为C:/Users/lenovo/.ssh） 的 id_rsa.pub文件打开，将内容全部复制；如果找不到该文件，设置查看文件的方式


  1. 直接在上面的命令框中输入 cat id_rsa.pub ，将内容直接复制

–> 将获取的 SSH 放到 GitHub 中
–> 点击 setting，然后点击 SSH and GPG keys，再点击 New SSH key 按钮




–> Title 中最好写一些内容，将复制的内容直接粘贴到 key 部分，再点击 Add SSH key

在 Git Bash 中检测 GitHub 公钥设置是否成功，cmd命令框中输入 ssh git@github.com如下则说明成功。



这里之所以设置 GitHub 密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在 GitHub 上，私钥放置在自己的电脑里。GitHub 要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了 ssh，当你推送的时候，git 就会匹配你的私钥跟 GitHub 上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。
